# Clear Progress

**GTD-приложение для доведения дел до конца**

## ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

**Версия 1.0**  
**Март 2026**

---

## 1. Общая информация

| Поле | Значение |
|------|----------|
| **Название проекта** | Clear Progress |
| **Тип приложения** | PWA (Progressive Web App) |
| **Назначение** | Персональное GTD-приложение для управления задачами и целями |
| **Целевая платформа** | Браузер (мобильный и десктоп), установка на домашний экран |
| **Архитектура** | PWA frontend + Google Apps Script backend + Google Sheets хранилище |

**Референсное приложение:** Maxdone (iOS/Android). Clear Progress развивает концепцию Maxdone с адаптацией под PWA-архитектуру и собственные доработки.

## 3. Детальное описание

### 3.1. Коробки задач (F01)

Основная навигационная модель приложения. Все задачи распределены по четырём временным горизонтам:

- **Сегодня** — задачи, которые нужно сделать сегодня
- **Неделя** — задачи на текущую неделю
- **Позже** — задачи без привязки к конкретному сроку (аналог Someday/Maybe в GTD)
- **Входящие (Inbox)** — точка сбора всех новых мыслей и задач для последующей обработки

Каждая коробка отображает количество задач в скобках рядом с названием. Экран задач показывает коробки последовательно с возможностью прокрутки.

### 3.2. Создание и редактирование задач (F02)

Задача — основная сущность приложения. При создании пользователь указывает название задачи и опционально заполняет дополнительные поля. Набор полей при создании настраивается (см. F18).

**Свойства задачи:**

- **Название** (обязательное)
- **Заметки** — текстовое поле для дополнительной информации
- **Цель** — привязка к одной из целей пользователя
- **Контекст** — GTD-контекст выполнения
- **Категория** — сфера жизни / область ответственности
- **Коробка** — в какой временной горизонт попадает задача

### 3.3. Цели (F05)

Цели — это многошаговые результаты, которых хочет достичь пользователь. Каждая цель объединяет под собой набор задач, необходимых для её достижения.

**Свойства цели:**

- Название и описание
- Картинка-обложка (визуальная идентификация)
- Статус: отменена, на паузе, не начата, в работе, завершена
- Счётчик привязанных задач (активных / всего)

Экран цели (F12) показывает шапку с обложкой, текущий статус и список привязанных задач, разделённых по коробкам.

### 3.4. Контексты и категории (F06, F07)

Контексты отражают обстоятельства выполнения задачи: место, инструмент, человек (@Дом, @Офис, @Телефон, @Компьютер). Категории группируют задачи по сферам ответственности (Работа, Семья, Здоровье, Хобби).

Оба справочника работают одинаково: список элементов с количеством привязанных задач, создание через интерфейс «Найти или создать», отдельный экран просмотра задач по выбранному элементу.

### 3.5. Чеклисты (F08)

Внутри задачи можно создать чеклист — список подпунктов, каждый из которых отмечается как выполненный или нет. Чеклист разделён на секции «Активно» и «Готово». В карточке задачи переключение между вкладками «Детали» и «Чеклист». Отображается счётчик прогресса (выполнено / всего).

### 3.6. Повторяющиеся задачи (F09)

Для регулярных дел задаче назначаются правила повторения. При завершении такой задачи система автоматически создаёт новую копию.

**Поддерживаемые интервалы:**

- Ежедневно
- По дням недели
- Еженедельно
- Ежемесячно
- Настраиваемый интервал

### 3.7. Синхронизация и офлайн (F23, F24)

Данные хранятся на двух уровнях: клиентский кэш (IndexedDB через Dexie.js) и серверное хранилище (Google Sheets через Apps Script). При наличии сети выполняется синхронизация. При отсутствии сети доступен просмотр кэшированных данных.

**Подключение бэкенда:** пользователь копирует шаблон Google Apps Script к себе в аккаунт, деплоит как Web App, вставляет URL в настройки PWA. Авторизация выполняется на уровне Apps Script (доступ только владельцу скрипта).

### 3.8. Акцентный цвет интерфейса (F22)

Пользователь выбирает основной цвет оформления из палитры. Цвет применяется к заголовкам, кнопкам, активным элементам, боковому меню и другим акцентным элементам.

**Доступные цвета:** зелёный (по умолчанию), оранжевый, фиолетовый, жёлтый, малиновый. Набор цветов может расширяться. Переключение доступно в любой момент через настройки.

## 4. Исключённый функционал

Следующие функции, присутствовавшие в референсном приложении Maxdone, исключены из Clear Progress:

| Функция | Причина исключения |
|---------|-------------------|
| **Даты задач (начало, дедлайн)** | Противоречит концепции коробок: задачи управляются временным горизонтом (Сегодня/Неделя/Позже), а не конкретными датами |
| **Вид «Календарь»** | Без дат задач календарный вид теряет смысл. Для привязки к датам используется внешний календарь |
| **Скрытие задач** | Усложняет модель без значительной пользы. Задачи управляются через коробки и завершение |
| **Календарные события** | Выходит за рамки GTD-приложения. Для событий используется Google Calendar или аналоги |
| **Шаблоны целей** | Избыточная сложность для MVP. Может быть добавлена в будущих версиях при наличии спроса |
| **Добавление через email** | Требует серверной инфраструктуры для приёма почты. Не вписывается в архитектуру PWA + Apps Script |
