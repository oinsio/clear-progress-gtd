**GTD-приложение для доведения дел до конца**

## Стек технологий

---

|---|---|

---

## 2. Архитектура

Приложение состоит из трёх компонентов. Каждый пользователь разворачивает собственный экземпляр бэкенда в своём Google-аккаунте. Фронтенд — единый для всех.

| Слой | Технология | Назначение |
|---|---|---|
| Frontend | React + TypeScript + Vite | PWA интерфейс пользователя |
| Backend / API | Google Apps Script (TypeScript через clasp) | REST API в Google-аккаунте пользователя |
| Хранилище данных | Google Sheets | БД в Google-аккаунте пользователя |
| Офлайн-кэш | IndexedDB (Dexie.js) | Локальный кэш для просмотра офлайн |

---

## 3. Frontend

### 3.1. Основные технологии

| Технология | Версия | Назначение |
|---|---|---|
| React | 18+ | Основной UI-фреймворк |
| TypeScript | 5+ | Типизация и качество кода |
| Vite | 5+ | Сборка и dev-сервер |
| Tailwind CSS | 3.4+ | Утилитарный CSS-фреймворк |
| shadcn/ui | latest | UI-компоненты (копируются в проект, полный контроль) |
| React Router | 6+ | Маршрутизация SPA |
| Dexie.js | 3+ | Обёртка над IndexedDB |
| vite-plugin-pwa | latest | Service Worker, манифест, кэширование |
| Vitest | 3+ | Unit-тестирование |
| @vitest/coverage-v8 | latest | Отчёты покрытия кода |

### 3.2. PWA-возможности

- Установка на домашний экран (Add to Home Screen)
- Service Worker для кэширования статики и данных (Workbox)
- Офлайн-режим: просмотр данных из IndexedDB при отсутствии сети
- Адаптивный дизайн (responsive) для всех размеров экрана

### 3.3. Хостинг фронтенда

Cloudflare Pages (основной вариант) или GitHub Pages (альтернатива). Оба бесплатны для статических сайтов, поддерживают HTTPS и кастомные домены. Cloudflare предпочтительнее за счёт встроенного CDN и автодеплоя из Git.

### 3.4. Тестирование

Vitest — основной фреймворк для unit-тестирования. Нативно интегрируется с Vite и переиспользует конфигурацию проекта (`vite.config.ts`), что обеспечивает нулевую настройку для TypeScript, path aliases и других параметров сборки.

Возможности:

- Встроенные моки и шпионы (`vi.fn()`, `vi.spyOn()`)
- Data-driven тесты (`it.each`) для параметризованных сценариев
- Watch-режим с горячей перезагрузкой при изменении кода
- Покрытие кода через `@vitest/coverage-v8`

Область покрытия: бизнес-логика, утилиты, сервисы синхронизации, преобразования данных. Тестирование UI-компонентов (React Testing Library, jsdom) — при необходимости в будущем.

## 5. Хранение данных

### 5.1. Серверное хранилище

Google Sheets в Google-аккаунте пользователя. Доступ через встроенный SpreadsheetApp API из Apps Script. Каждая сущность GTD (задачи, проекты, контексты и т.д.) — отдельный лист (sheet) в файле.

### 5.2. Клиентский кэш

IndexedDB (через Dexie.js) в браузере пользователя. При наличии сети данные синхронизируются с Google Sheets. При отсутствии сети пользователь может просматривать последнюю закэшированную копию данных.

---

## 6. Модель дистрибуции

Фронтенд размещается централизованно на хостинге. Бэкенд устанавливается каждым пользователем самостоятельно.

### 6.1. Процесс установки для пользователя

1. Открыть PWA в браузере (или установить на домашний экран)
2. Скопировать шаблон Google Apps Script к себе в Google-аккаунт
3. Задеплоить скрипт как Web App
4. Вставить URL своего Web App в настройки PWA

### 6.2. Перспектива

В будущем возможен переход на гибридную модель: минимальный Google Workspace Add-on в Marketplace для автоматической настройки бэкенда, а основной интерфейс остаётся в PWA.

## 7. Схема взаимодействия

```
Пользователь (браузер / PWA)
│
▼
React + TypeScript + Vite + Tailwind + shadcn/ui + Dexie.js
│ (есть сеть → REST API)
▼
Google Apps Script Web App (экземпляр пользователя)
│ (SpreadsheetApp API)
▼
Google Sheets (аккаунт пользователя)
```

---

## 8. Преимущества выбранного стека

- Единый язык (TypeScript) на всех уровнях — упрощает вайб-кодинг и поддержку
- Нулевая стоимость инфраструктуры — все компоненты бесплатны
- Полный контроль пользователя над своими данными — всё хранится в его Google-аккаунте
- React + Tailwind + shadcn/ui — отличная поддержка в AI-кодогенерации, компоненты хранятся в проекте и полностью контролируемы
- Офлайн-просмотр данных через IndexedDB + Service Worker
- Простая установка бэкенда для конечного пользователя (3–4 шага)
